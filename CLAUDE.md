## CRITICAL RULES (NON-NEGOTIABLE)

### Command Execution Policy

DO:
- Use `./bin/magnum <command>` for ALL operations

NEVER:
- Run npm, vite, wrangler, or eslint directly
- Bypass the `./bin/magnum` wrapper in any way

Violating this policy is treated as a broken build.

### Available Commands

```
./bin/magnum start           # Build and start the frontend container
./bin/magnum stop            # Stop the stack
./bin/magnum restart         # Stop, rebuild, and start
./bin/magnum logs [-f]       # View container logs
./bin/magnum exec [cmd]      # Shell into the container
./bin/magnum lint            # Run ESLint inside the container
./bin/magnum build           # Production build (nginx stage)
./bin/magnum help            # Show all commands
```

### Git Commit Message Policy

NEVER use `cat`, heredoc (`<<EOF`), or any pipe/subshell to pass commit messages.

ALWAYS pass the message directly with `git commit -m "message"`.

CORRECT:
```
git commit -m "feat: Add contact form validation"
```

FORBIDDEN:
```
git commit -m "$(cat <<'EOF'
feat: Add contact form validation
EOF
)"
```

### Deprecated Code Policy

NEVER:
- Use deprecated functions, classes, modules, or API endpoints
- Leave deprecated code in the codebase
- Introduce new occurrences of deprecated patterns

If you find deprecated code, replace it immediately.

## Coding Standards

### React / JSX Style

- **Components**: Functional components with hooks only (no class components)
- **File naming**: PascalCase for component files (`Hero.jsx`, `Contact.jsx`)
- **Exports**: Default export for page/component files
- **Props**: Destructure in function signature
- **State**: Use `useState` / `useReducer` for local state
- **Effects**: Clean up side effects in `useEffect` return

### Naming Conventions

| Context | Convention | Example |
|---|---|---|
| Components | PascalCase | `BookCallFloater` |
| Functions / variables | camelCase | `sendContactMessage` |
| CSS classes | kebab-case (Tailwind) | `text-primary-foreground` |
| Constants | UPPER_SNAKE_CASE | `API_BASE_URL` |
| Files (components) | PascalCase | `Hero.jsx` |
| Files (utilities) | camelCase | `sendContactMessage.js` |

### Clean Code Principles

Complexity and Readability:
- Reduce cognitive complexity — break down complex logic into smaller, focused functions
- Use guard clauses to return early and reduce nesting
- Maximum nesting depth: 2 levels
- Keep variables tightly scoped — declare close to where they are used

Code Organisation:
- One responsibility per function / component
- Related functions grouped together
- Helper functions defined close to where they are used
- Use meaningful names that describe intent

Component Patterns:
- Extract repeated JSX into sub-components
- Keep component files focused — one primary export per file
- Separate data-fetching logic from presentation
- Use composition over prop drilling

## Docker Development

The project uses Docker for local development to avoid platform-specific issues with Cloudflare's `workerd` runtime.

- `DISABLE_CLOUDFLARE_PLUGIN=true` is set in Docker to skip the `cloudflare()` Vite plugin (workerd cannot run inside containers; this only affects serverless API functions, not the landing page)
- Source code is bind-mounted for hot reloading
- `node_modules` uses a named volume to avoid host/container binary conflicts
